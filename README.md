# Введение
Сорвенование представляет из себя создание системы автоматического распознавания речи (ASR), которая сможет точно транскрибировать произносимые числа от 1000 до 999 999 на русском языке.

Обучаемые модели должны быть < 5M параметров.

Набор данных содержит аудиозаписи того, как несколько дикторов (голосов) произносят цифры на русском языке. Например, для файла с целевой транскрипцией 280,520 в аудиозаписи есть произношение строки "двести восемьдесят тысяч пятьсот двадцать".
<details>
<summary>Подробное описание данных</summary>

Данные разделены на 3 выборки:

* train - данные для обучения 
* dev - данные для валидации
* test - данные для тестирования (неразмеченные)

В наборе данных представлены две подгруппы голосов:

* Внутренние динамики (inD) — spk_id, которые отображают обучающий набор голосов
* Внешние динамики (ooD) - spk_id, которые представляют невидимый набор голосов.

Подробная информация об аудиофайле:
* Формат: .wav, .mp3
* Частота дискретизации: 16, 22,05, 24 кГц
* Количество каналов: 1 (моно)
* Длительность аудио (секунды):
* среднее значение: 3,20
* std: 0,56
* минимальное значение: 1,40
* 25%: 2.80
* 75%: 3.59
* макс.: 5,55
</details>

Целевая метрика:

$$
\mathrm{Score} = \frac{2 \cdot \mathrm{CER}_{\mathrm{inD}} \cdot \mathrm{CER}_{\mathrm{ooD}}}{\mathrm{CER}_{\mathrm{inD}} + \mathrm{CER}_{\mathrm{ooD}}} \cdot 100\%
$$
<details>
<summary>Ограничения на модель</summary>

* Учитывая текстовую область, допускается использование языковой модели (LM) для декодирования, слияния и повторного отображения (с n-граммовыми языковыми моделями, подобными KenLM).
* Уделите особое внимание надежности: аудиозаписи при тестировании содержат некоторые признаки ухудшения качества сигнала в реальных условиях, которые могут включать потерю пакетов, фоновый шум и другие факторы
* Обучение модели может проводиться в автономном режиме независимо от тестовых записных книжек, хотя для проведения оценки и подготовки заявки требуются обучающие артефакты
* Обученная модель должна обрабатывать входной аудиосигнал частотой 16 кГц для целей оценки
</details>

## Пайплайн 1 (основной)
1. Числа <-> Текст<br>
Прямое преобразование (int_to_russian_words):
Поддерживает числа от 0 до 999 999. Учитывает род и падеж для «тысяча» (одна тысяча, две тысячи, пять тысяч).
Используются словари для единиц (в мужском и женском роде), десятков, сотен и исключений 10–19.
Обратное преобразование (words_to_int):
Разбирает фразу вида «двадцать три тысячи четыреста пять» -> 23405.
Ищет маркер «тысяча/тысячи/тысяч», парсит тысячные и остаточные части через словари.

2. Аудиопредобработка<br>
Ресемплинг до 16 кГц, стерео -> моно.
Мел-спектрограмма: n_mels=80, n_fft=400, hop_length=160 (10 мс при 16 кГц), win_length=400 (25 мс).
Логарифмирование (log(mel + 1e-6)).
Выход: тензор формы (время, n_mels).

3. Аугментации сырых аудио
* Наложение случайного шума с заданным SNR (5–20 дБ). При отсутствии внешних шумов не применяется;
* Изменение темпа (коэффициенты 0.85, 0.95, 1.0, 1.05, 1.15) через ресемплинг туда-обратно;
* Случайное изменение громкости (диапазон –20 … +20 дБ);
* Сдвиг высоты тона на -3 … +3 полутона.
Порядок применения: pitch -> speed -> volume -> noise.

4. Аугментации мел-спектрограмм
* SpecAugment (spec_augment): 3 временных маски (ширина до 15 фреймов) и 3 частотных маски (ширина до 12 мел-каналов);
* Заливка нулями;
* Feature dropout:с вероятностью 20% обнуляется случайный непрерывный блок частотных каналов (ширина ~10% от n_mels).

5. Глобальная нормализация мел-спектрограмм<br>
Класс GlobalMelNormalizer вычисляет среднее и стандартное отклонение по всем обучающим примерам (собирает все пиксели всех спектрограмм).
При нормализации из каждого элемента вычитается глобальное среднее и делится на глобальный стандарт (стабильнее, чем instance-нормализация по каждому файлу).

6. Датасет (RussianNumbersDataset)<br>
Загружает CSV с колонками filename и transcription.
Для каждого файла: загрузка аудио -> применение аугментаций -> получение mel-> SpecAugment -> глобальная нормализация -> feature dropout.
Текст числа преобразуется в словесную форму (int_to_russian_words) и затем в последовательность индексов символов по словарю vocab.
Возвращает: (mel_tensor, target_indices, mel_length, target_length).
Collate_fn паддит мел-спектрограммы и целевые последовательности до максимума в батче.
7. Архитектура модели SimpleASRModel<br>
Сверточная часть (извлечение признаков во времени и частоте):
* Conv2d(1->32, stride=(2,2)) + BN + ReLU + Dropout2d(0.2)
* Conv2d(32->64, stride=(2,2)) + BN + ReLU + Dropout2d(0.2)
* Conv2d(64->128, stride=(1,2)) + BN + ReLU<br>
Итоговое сжатие частотной оси в 8 раз (80 -> 10 частотных бинов)<br>
Рекуррентная часть:<br>
* Dropout(p=0.4) перед GRU для регуляризации
* Двунаправленный GRU: 2 слоя, hidden_dim=128 -> выход 256
* Еще один Dropout(0.4) после GRU<br>
Выходной слой:
* Linear(256 -> num_classes=41 (blank + пробел + 33 буквы + ё))

8. Обучение и инференс<br>
* Количество эпох = 30
* CTCLoss - лосс-функция
* Оптимизатор: AdamW (lr=5e-4, weight_decay=1e-3) – адаптивная регуляризация;
* Планировщик: CosineAnnealingLR на 30 эпох, минимальная скорость 1e-5;
* Gradient clipping (norm=1.0);
* Early stopping (Отсутствие улучшения в течение 10 эпох -> остановка обучения).<br>
Сохраняется лучшая модель.<br>

В качестве метода декодирования использовался Greedy Decoder. В ходе экспериментов Greedy Decoder показал себя лучше чем Beam search Shallow Fusiuon, поэтому финальный сабмит генерировался именно с помощью Greedy.

## Пайплайн 2
Был проведен еще один эксперимент (топ-2 сабмит CER = 14.510):
1. Подготовка данных и LM<br>
Генерация текстового корпуса (1000 -> "тысяча", 999999 -> "девятьсот девяносто девять тысяч...")
Обучение 3-gram KenLM на корпусе -> numbers.arpa
Словарь: 33 рус. буквы + пробел + < blank > (CTC)
2. Архитектура модели<br>
Audio -> MFCC(40) -> CNN (Conv1d -> 4x ResBlock** -> downsample ×4 по времени) -> BiLSTM -> Linear
3. Feature Engineering<br>
* MFCC: 40 коэффициентов, n_fft=400, hop=160 (10ms шаг);
* Нормализация: Instance Norm по временной оси
* Длины: feat_lens = wave_lens // 160 (учёт downsampling)
4. Обучение и инференс
* Количество эпох = 10;
* CTCLoss - лосс-функция;
* Оптимизатор: AdamW (lr=5e-4, weight_decay=1e-4) – адаптивная регуляризация;
* Планировщик: CosineAnnealingLR на 30 эпох, минимальная скорость 1e-5;
* Gradient clipping (norm=1.0);

 ** ResBlock: Conv1d(kernel_size=5) -> Conv1d(kernel_size=1) -> BatchNorm -> BatchNorm -> GeLU -> Dropout(p=0.1)

В качестве метода декодирования использовался использовался метод Beam search Shallow Fusiuon
